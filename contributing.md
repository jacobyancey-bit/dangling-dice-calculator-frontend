# Contributing to Dangling Dice Calculator Frontend

Thank you for your interest in contributing! We welcome contributions of all kinds.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/dangling-dice-calculator-frontend.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes: `flutter test`
6. Commit with clear messages: `git commit -m "Add feature: description"`
7. Push to your branch: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Code Guidelines

- Follow [Dart Style Guide](https://dart.dev/guides/language/effective-dart/style)
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and small
- Use const constructors where possible

## Testing

- Write unit tests for new functions and services
- Write widget tests for new UI components
- Run `flutter test` before submitting a PR
- Aim for at least 80% code coverage

## Commit Messages

Use clear, descriptive commit messages:

- `feat: Add calculator history screen`
- `fix: Resolve decimal input validation bug`
- `refactor: Simplify calculation service`
- `docs: Update README with API docs`
- `test: Add tests for expression parser`

## Pull Request Process

1. Update the README.md with any new features or changes
2. Ensure all tests pass: `flutter test`
3. Request a review from maintainers
4. Address feedback and make requested changes
5. Merge once approved

## Areas for Contribution

- Bug fixes
- UI/UX improvements
- Performance optimization
- Documentation
- Tests and test coverage
- Accessibility improvements
- Localization (i18n)

## Questions?

Feel free to open an issue to discuss potential changes before submitting a PR.

Thank you for contributing! 🎉
